# OhmyHotel 2.0 — Product Requirements Document (PRD)
**Version:** 1.0.0  
**Date:** 2026-04-14  
**Owner:** OMH Global Strategy  
**Status:** Ready for Development

## 1. 프로덕트 비전

### 1.1 미션
> "한국인을 위한 아시아 최고의 여행 예약 플랫폼"  
> 글로벌 OTA가 채우지 못하는 한국인 맞춤 여행 경험을 제공한다.

### 1.2 포지셔닝
- **타깃:** 일본·동남아 여행을 즐기는 한국인 (20~50대)
- **핵심 차별점:** KRW 투명 가격 + 완전 한국어 최적화 + 아시아 특화 큐레이션
- **경쟁 우위 확보 축:** 신뢰(리뷰) + 전환(희소성 UX) + 재방문(로열티) + 수익(교차판매)

### 1.3 성공 지표 (KPI)
| 지표 | 현재 (추정) | 6개월 목표 | 12개월 목표 |
|------|------------|-----------|------------|
| 검색→예약 전환율 | ~2% | 3.5% | 5%+ |
| 재방문율 (30일) | ~10% | 20% | 35% |
| 리뷰 보유 호텔 비율 | 0% | 60% | 90% |
| OMY Points 가입률 | 0% | 25% | 50% |
| 평균 객단가 (ARPU) | 기준값 | +15% | +30% |

## 2. 사용자 여정 (Full Booking Flow)

```
[홈] → [자동완성 검색] → [검색결과 + 필터] → [호텔 상세] 
  → [객실 선택] → [여행자 정보] → [결제] → [예약 확인]
```

총 7단계 예약 플로우. 각 단계마다 이탈 방지 장치 탑재.

## 3. 기능 요구사항 (Feature Requirements)

### FR-01: 메인 홈 / 검색 진입

#### FR-01-1: 스마트 자동완성 검색
- **설명:** 사용자가 목적지 입력 시 실시간으로 도시/호텔명/랜드마크 제안
- **우선순위:** P0 (즉시)
- **세부 요건:**
  - 입력 시작 1글자부터 디바운스 300ms 후 제안
  - 한국어/영어/일본어 동시 입력 지원
  - 오타 허용 (Fuzzy search: "토쿄" → "도쿄" 자동 교정)
  - 제안 카테고리: 도시 / 호텔 직접 / 인기 지역 / 최근 검색
  - 각 제안에 해당 지역 숙소 수 표시 ("도쿄 · 숙소 3,241개")
  - 최근 검색 기록 5개 저장 (localStorage)
  - 인기 검색 지역 상단 고정 (서버에서 관리)
- **데이터 소스:** Elasticsearch 인덱스 (도시, 호텔명, 랜드마크)
- **성능:** 200ms 이내 응답

#### FR-01-2: 날짜 선택 (달력)
- **설명:** 체크인/체크아웃 날짜 선택 UI
- **세부 요건:**
  - 날짜별 최저가 표시 (KRW, 해당 목적지 기준)
  - 성수기/비수기 색상 구분 (빨강=고가, 초록=저가)
  - 최소 1박 / 최대 30박
  - 과거 날짜 선택 비활성화
  - 선택된 날짜 범위 하이라이트

#### FR-01-3: 인원 선택
- 성인 (만 18세 이상): 1~4명
- 아동 (만 2~17세): 0~3명, 나이 선택 필수
- 룸 수: 1~3개

#### FR-01-4: 홈 큐레이션 컨텐츠
- **오마이 픽:** 에디터 추천 호텔 (수동 관리, CMS)
- **기획전 배너:** 시즌별 프로모션 (관리자 설정)
- **지역별 인기 목적지:** 클릭 시 해당 지역 검색 결과로 이동
- **최근 본 호텔:** 로컬스토리지 기반, 최대 5개

### FR-02: 검색결과 리스트

#### FR-02-1: 호텔 카드
각 호텔 카드에 표시되는 정보:
- 호텔 대표 이미지 (lazy loading)
- 호텔명 (한국어 우선, 없으면 영문)
- 성급 (별 아이콘)
- 위치 (지역명 + 지도 아이콘)
- 리뷰 점수 (0~10, 소수점 1자리) + 리뷰 수
- 리뷰 태그 (상위 2개: "위치 최고", "청결도 우수" 등)
- 1박 최저가 (KRW, 세금 포함 여부 명시)
- 할인율 배지 (프로모션 적용 시)
- **FOMO 배지 (복수 표시):**
  - 🔴 "오늘 이 날짜 객실 N개 남음" (잔여 5개 이하)
  - 🟠 "방금 X분 전 예약됨"
  - 🟡 "이번 주 N번 예약"
  - 🟢 "무료 취소 가능"
- OMY Points 적립 예상액 ("예약 시 ♦ 450P 적립")
- 즐겨찾기 하트 버튼

#### FR-02-2: 필터 시스템
**상단 빠른 필터 칩 (자주 사용):**
- 무료 취소 가능
- 조식 포함
- 수영장 있음
- 4성급 이상

**상세 필터 패널:**
- 가격대: 슬라이더 (₩0 ~ ₩1,000,000)
- 성급: 1~5성 체크박스
- 리뷰 점수: 8.0+, 7.0+, 6.0+
- 숙소 유형: 호텔, 리조트, 게스트하우스, 에어비앤비형
- 시설: 수영장, 헬스장, 조식, 주차, 반려동물, 조식뷔페
- 취소 정책: 무료 취소 가능

#### FR-02-3: 정렬
- 추천순 (기본, 전환율 최적화 알고리즘)
- 가격 낮은순 / 높은순
- 리뷰 점수 높은순
- 거리순 (랜드마크 기준)

#### FR-02-4: 지도 뷰
- 호텔 위치 마커 + 가격 표시
- 마커 클릭 시 미니 카드 팝업

### FR-03: 호텔 상세 페이지

#### FR-03-1: 사진 갤러리
- 메인 이미지 + 썸네일 그리드
- 전체보기 시 라이트박스 슬라이더
- 카테고리별 분류: 객실, 수영장, 식당, 로비, 주변

#### FR-03-2: 호텔 기본 정보
- 호텔명, 성급, 체인 브랜드
- 주소 (한국어 표기) + 지도 연동
- 체크인/아웃 시간
- 시설 아이콘 그리드
- 수상 배지 (있는 경우)

#### FR-03-3: 리뷰 시스템 (FR-REVIEW)
- **종합 점수:** 10점 만점, 대형 숫자로 강조
- **카테고리별 점수:**
  - 청결도 / 위치 / 직원 친절도 / 시설 / 가격 대비 만족도
  - 진행 바 형태로 시각화
- **리뷰 목록:**
  - 별점, 작성자 국적 (국기 아이콘), 작성일
  - 여행 유형 (커플/가족/비즈니스/솔로)
  - 리뷰 본문 (200자 초과 시 "더보기" 접기)
  - 긍정/부정 키워드 태그
- **리뷰 필터:** 전체 / 한국인만 / 최근순 / 평점순
- **리뷰 요약 AI:** 상위 키워드 자동 추출 (Phase 2)

#### FR-03-4: 고정 CTA 바
- 스크롤해도 화면 하단에 고정
- 최저가 표시 + "객실 선택하기" 버튼
- 잔여 객실 경고 문구 포함

### FR-04: 객실 선택

#### FR-04-1: 객실 타입 카드
각 객실 타입별:
- 객실명 (한국어)
- 대표 이미지
- 면적 (㎡)
- 최대 인원
- 시설 아이콘 (침대 타입, 금연/흡연, 창문 뷰)
- **옵션 그룹 (동일 타입, 조건별 가격 비교):**

| 옵션 | 가격 | 취소정책 | 조식 |
|------|------|----------|------|
| 기본 | ₩120,000 | 환불불가 | 미포함 |
| 무료취소 | ₩145,000 | 무료취소 | 미포함 |
| 조식포함 | ₩168,000 | 환불불가 | 포함 |
| 무료취소+조식 | ₩190,000 | 무료취소 | 포함 |

- 잔여 객실 수 표시 (3개 이하 시 강조)
- "마지막 1개!" 배지

#### FR-04-2: 옵션 선택 UX
- 무료취소/환불불가 차이 툴팁 설명
- 조식 내용 팝업 (메뉴 정보)
- 선택된 옵션 우측에 요약 표시

### FR-05: 여행자 정보 입력

#### FR-05-1: 진행 단계 바
상단 고정: `①객실선택 → ②여행자정보 → ③결제`

#### FR-05-2: 입력 폼
- 대표 투숙객 성명 (영문, 여권 기준)
- 연락처 (한국 휴대폰 형식 우선)
- 이메일 (예약 확인서 발송용)
- 특별 요청사항 (텍스트 박스, 선택)
- 도착 예정 시간 (선택)

#### FR-05-3: 예약 요약 사이드바 (데스크탑) / 하단 바 (모바일)
- 호텔명 + 객실 타입
- 체크인/아웃 날짜, 박수
- 인원
- **요금 분해 표시:**
  - 객실 요금 (N박 × ₩XXX,XXX)
  - 세금 및 봉사료
  - 쿠폰/포인트 할인
  - **최종 결제 금액 (강조)**
- 무료 취소 기한 재확인 문구

### FR-06: 결제

#### FR-06-1: 결제 수단
**1단계 목표:**
- 신용/체크카드 인라인 폼 (NICEPAY → 인라인 전환)
  - 카드번호, 유효기간, CVC 직접 입력
  - 지원 카드: KB, 신한, 삼성, 하나, 현대, BC, 농협, 우리
- 카카오페이
- 네이버페이

**2단계 목표 (Phase 2):**
- 토스페이
- Apple Pay / Google Pay
- Alipay (중화권 고객 대응)
- PayPal

#### FR-06-2: OMY Points 사용
- 로그인 사용자: 보유 포인트 표시
- 포인트 부분 사용 가능 (1P = ₩1)
- 최대 결제금액의 20% 사용 가능

#### FR-06-3: 결제 직전 안심 메시지
- 무료취소 기한 최종 확인
- 해당 호텔 최고 평가 리뷰 한 줄 노출
- 보안 배지 (SSL, PCI-DSS)
- "5,000명이 이미 예약한 호텔" 등 사회적 증거

#### FR-06-4: 환불불가 상품 경고
- 별도 확인 팝업 (기존 UX 유지 + 강화)
- "환불불가 상품입니다. 계속 진행하시겠습니까?" 명시적 동의

#### FR-07-1: 예약 완료 정보
- 예약번호 (굵게 강조)

- 앱 다운로드 유도 배너 (예약 완료 = 만족감 최고 시점)

### FR-08: OMY Points (로열티 프로그램)

#### FR-08-1: 적립 구조
- 예약 확정 시: 결제금액의 1% 적립 (1P = ₩1)
- 리뷰 작성 시: 200P 추가 적립
- 생일 달: 2배 적립
- 첫 예약: 1,000P 웰컴 보너스

#### FR-08-2: 등급 체계
| 등급 | 조건 (연간) | 혜택 |
|------|------------|------|
| 블루 | 기본 | 1% 적립 |
| 실버 | 300,000원 이상 | 1.5% 적립 + 무료 객실 업그레이드 요청 가능 |
| 골드 | 1,000,000원 이상 | 2% 적립 + 레이트 체크아웃 + 웰컴 어메니티 |
| 다이아 | 3,000,000원 이상 | 3% 적립 + 전용 CS + 공항 라운지 쿠폰 |

#### FR-08-3: 포인트 만료
- 마지막 적립/사용 후 2년
- 만료 30일 전 이메일 알림

### FR-09: 비기능 요구사항

#### FR-09-1: 성능
- 첫 화면 LCP(Largest Contentful Paint): 2.5초 이내
- 검색 자동완성 응답: 200ms 이내
- 검색결과 로딩: 1초 이내 (스켈레톤 UI 제공)
- 결제 처리: 5초 이내

#### FR-09-2: 반응형 (Responsive)
- 모바일 우선 설계 (Mobile-First)
- 브레이크포인트: 375px / 768px / 1280px / 1440px+
- 터치 최적화 (최소 터치 영역 44×44px)

#### FR-09-3: 접근성
- WCAG 2.1 AA 기준
- 스크린리더 호환

#### FR-09-4: 보안
- PCI-DSS 준수 (카드 정보 서버 비저장)
- HTTPS 강제
- CSRF 토큰
- Rate Limiting (검색 API: 100req/min/IP)

#### FR-09-5: SEO
- Next.js SSR/SSG로 검색 노출 최적화
- 구조화 데이터 (Schema.org: Hotel, Offer, Review)
- Open Graph 메타태그 (SNS 공유 최적화)

## 4. 비즈니스 규칙

### BR-01: 가격 표시
- 모든 가격은 KRW 기본 표시
- 세금포함/세금별도 여부 명시 필수
- 세금 = 숙박세 + 봉사료 (호텔마다 상이)
- 표시 규칙: "₩120,000 (세금 포함)" 또는 "₩120,000 + 세금 ₩12,000"

### BR-02: 취소 정책 표시
- 무료취소 가능 시: 취소 마감일시 명시 "2026.04.17 14:00까지 무료 취소"
- 환불불가: 굵은 빨간 텍스트 + 확인 팝업
- 부분 환불: 환불 금액 및 기준 명시

### BR-03: FOMO 배지 조건
- "X개 남음" 배지: 잔여 5개 이하 실시간 재고 연동
- "N분 전 예약" 배지: 실제 예약 데이터 기반 (조작 금지)
- "이번 주 N번 예약": 실제 집계 데이터 기반

## 6. 용어 정의

| 용어 | 정의 |
|------|------|
| OMY Points | OhmyHotel 자체 로열티 포인트 (1P = ₩1) |
| FOMO 배지 | 희소성·긴급성을 나타내는 UI 요소 |
